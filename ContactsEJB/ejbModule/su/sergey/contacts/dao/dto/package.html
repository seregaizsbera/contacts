Структура автогенерирующихся DAO и сопутствующих классов

Для каждой таблицы генерируются следующие классы

  • TableDAO
  • TableData
  • TableCreateInfo
  • TableUpdateInfo

Здесь Table заменяется некоторым префиксом. Префикс для каждой таблицы выбирается в ручную и никак не связан именем таблицы. Имена свойств образуются из имен колонок по определенному правилу.

TableDAO

Этот класс содержит методы для работы с соответствующей таблицей. Он наследуется от AbstractDAO и не специфицирует ConnectionSource. Соответственно используется ConnectionSource по умолчанию. То есть эти DAO можно использовать только из контейнера, и в контексте использующего их модуля под JNDI именем ContainerConnectionSource.DEFAULT_DATA_SOURCE должен быть сконфигурирован DataSource, настроенный на соответствующую базу. Экземпляры данного класса получают по паттерну singleton.

public <?> create(TableCreateInfo value) throws DAOException

Создает запись в соответствующей таблице, на основе указанных значений. В случае наличия у данной таблицы автогенерируещегося ключа возвращает его.

public TableData find(TableData value) throws DAOException

По первичному ключу, заданному в параметре пытается найти соответствующую запись и вставить данные из неё в соответствующие свойства аргумента. Возвращает аргумент, заполненный данными, или null, в случае если запись не была найдена по первичному ключу.

public void update(<?>, TableUpdateInfo value) throws DAOException

По значениям первых аргументов, которые соответствуют полям первичного ключа этой таблицы, проводит обновление остальных полей таблицы указанными значениями.

public void remove(<?>) throws DAOException

Удаляет запись в таблице по указанному первичному ключу.

public void addOuts(SqlOutAccessor accessor)

С помощью интерфейса SqlOutAccessor добавляет поля таблицы ровно в том порядке, в котором их считывает метод populate.

public int populate(TableData value, ResultSet rs, int startIndex) throws SQLException

Читает данные из текущей строки rs в value используя в качестве начального индекса для считывания данных startIndex. Возвращает номер поля следующего за последним считанным.

TableData

Этот класс является простым контейнером, поля которого полностью повторяют поля соответствующий таблицы в базе данных.

TableCreateInfo

Этот интерфейс определяет get методы для получения данных, необходимых для создания новой записи в соответствующей таблице.

TableUpdateInfo

Этот интерфейс определяет get методы для получения данных необходимых и достаточных для обновления записи в соответствующей таблице.
